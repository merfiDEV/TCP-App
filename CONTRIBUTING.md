# Contributing to TCP Demo App

First off, thank you for considering contributing to TCP Demo App! It's people like you that make TCP Demo App such a great tool.

## Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our [Issues](../../issues) first. If it's not there, feel free to open a new one!

## Fork & create a branch

If this is something you think you can fix, then fork TCP Demo App and create a branch with a descriptive name.

## Get the Code
1. Clone the repository: `git clone https://github.com/iqfareez/TCP-App.git`
2. Open the solution `TCP-App.sln` in Visual Studio or JetBrains Rider.
3. Build the solution to restore NuGet packages (if any).

## Coding Standards

*   Please write clear, readable, and concise code.
*   Comment your code where necessary, especially for complex logic.
*   Follow standard C# naming conventions (PascalCase for classes/methods, camelCase for local variables).

## Commit your changes

Make sure your commit messages are descriptive and explain the changes you have made. We encourage using [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/).

## Pull Request

When you're ready, submit a Pull Request! Please provide a clear description of the problem you're solving or the feature you're adding. 

Your PR will be reviewed by maintainers, and we might suggest some changes or improvements. 

Thank you for your contribution!
